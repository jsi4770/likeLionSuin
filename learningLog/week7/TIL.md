### 2. 핵심 내용 요약

1. **자연어 처리(NLP)의 본질**
   - 자연어 처리는 인간의 언어를 수학적 숫자 배열로 변환하고, 그 숫자들 사이의 패턴을 학습하는 통계적 방법론이다.
   - 텍스트를 컴퓨터가 처리 가능한 형태로 분할하고 변환하는 전처리 과정이 필수적이다.

2. **텍스트를 숫자로 (전처리)**
   - **토큰화(Tokenization)**: 문장을 모델이 처리할 수 있는 최소 단위인 토큰으로 분할하는 작업이다. 현대 LLM은 의미를 가진 최소 단위로 분할하여 어휘 폭발 문제를 해결하는 '서브워드 토큰화'를 표준으로 채택한다.
   - **임베딩(Embedding)**: 토큰화된 조각들을 다차원 공간의 벡터(숫자 배열)로 변환하는 과정이다. 모델은 단어의 의미를 사전적으로 암기하는 대신, 함께 자주 등장하는 단어들을 벡터 공간에서 가깝게 배치한다.

3. **트랜스포머 (Transformer)**
   - **Self-Attention**: 현재 단어를 이해하기 위해 문장 내 어떤 단어에 가중치를 두어야 하는지를 계산하는 기술이다.
   - **문맥 이해**: 문장 전체를 동시에 펼쳐 놓고 단어 간 관계성을 계산하므로, 긴 뉴스 기사나 논문에서도 맥락을 정확히 포착할 수 있다.

4. **허깅페이스 (Hugging Face)**
   - 전 세계 연구자와 개발자가 모이는 표준 허브로, 복잡한 모델 구현을 효율적으로 돕는다.
   - **3대 구성**: 사전학습된 모델 저장소(Models), 학습용 데이터 저장소(Datasets), 웹 데모 실행 환경(Spaces)으로 구성된다.
   - **Pipeline**: 토큰화, 임베딩, 모델 추론의 전 과정을 단일 함수로 실행할 수 있도록 추상화한 인터페이스다.

### 3. 기술 비교

**RNN vs Transformer**

| 구분 | RNN (과거) | Transformer (현재) |
| :--- | :--- | :--- |
| **처리 방식** | 앞에서부터 순차적으로 단어 처리 | 문장 전체를 동시에 처리 |
| **한계/장점** | 문장이 길어질수록 앞부분 정보 소실 | 장거리 의존성 문제 해결 |
| **연산 효율** | 순차 처리로 병렬화 불가 | 병렬 연산 가능 |

### 4. 실습 / 사례 연구
- **Attention 적용**: "중앙은행이 기준 금리를 큰 폭으로 인하했다. 그것은 시장에 호재로 작용했다."라는 문장에서 '그것'을 처리할 때, 모델은 앞 문장의 '금리'와 '인하'에 높은 Attention Score를 부여한다.
- **벡터 연산**: '금리'와 '주가'는 벡터 공간에서 가깝게 배치되지만, '사과'나 '바나나'와 같은 단어는 멀리 배치되어 의미적 차이를 나타낸다.
